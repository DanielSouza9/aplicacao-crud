Escolhi o gerenciamento de clientes que se cadastraram para receber descontos em uma farmácia fictícia como tema para esse trabalho. Esse é um tema simples, mas é suficiente o bastante para mostrar o funcionamento das operações CRUD. 
Esse sistema funciona apresentando um menu para o usuário, atendente da farmácia, que poderá gerenciar o cadastro dos clientes desta farmácia, criando, alterando, mostrando ou excluindo um cadastro a qualquer momento que acessar o sistema.
Internamente, o sistema coleta, no cadastro, os dados do usuário (nome e email) e salva esses dados em um arquivo externo (json) que funciona como um banco de dados do código, e recebe também todas as informações dos clientes cadastrados, como ID, email e nome, o que permite que essas informações não se percam quando o usuário sair do sistema ou caso for feita alguma alteração no cadastro posteriormente.
Nesse código, ('Drogaria Eunápolis.py'), usei as operações CRUD para executar o gerenciamento de clientes. Fiz isso no código por meio de funções específicas que manipulam uma lista de dicionários, utilizada como estrutura de dados principal. A operação Create é realizada pela função criar, que adiciona novos clientes com ID, nome e email e salva os dados em um arquivo JSON. O Read ocorre pela função listar, que exibe todos os registros, e ler, que busca um cliente específico pelo ID. A operação Update é feita pela função atualizar, permitindo alterar nome e/ou email de um cadastro existente. Já o Delete é executado pela função deletar, que remove registros da lista usando o método remove(). As funções carregar e salvar fazem a conexão com o arquivo json garantindo a permanência dos dados, enquanto o menu interativo possibilita o usuário executar todas essas operações pelo terminal.
